권한·소유권·특수권한

chmod 숫자 계산, chown/chgrp, umask, setuid, setgid, sticky bit를 시험식으로 정리합니다.

권한

1. 단원 개요

파일과 디렉터리 접근은 소유자, 그룹, 기타 사용자 권한으로 통제되며 rwx를 숫자로 계산할 수 있어야 합니다.

관련 CBT 풀기 리눅스마스터 연계 보기

2. 기말고사에서 중요한 이유

기말과 리눅스마스터 모두 chmod 755/644, umask 022, setuid/setgid/sticky bit를 자주 묻습니다.

3. 리눅스마스터 2급 2차 기출 연계 포인트

  • rwx 숫자 계산
  • chmod와 chown 구분
  • umask 결과 계산
  • setuid/setgid/sticky bit 의미
  • 공유 디렉터리 삭제 권한 함정

4. 핵심 개념 설명

  • r=4, w=2, x=1입니다.
  • 권한은 user, group, others 순서로 표시됩니다.
  • 파일 기본 권한은 보통 666, 디렉터리는 777에서 umask로 제거한다고 이해합니다.
  • 특수 권한 앞자리 4는 setuid, 2는 setgid, 1은 sticky bit입니다.

5. 주요 명령어

명령어핵심 의미
chmod 755 file권한 숫자 방식 변경
chmod u+x script.sh기호 방식 권한 추가
chown user:group file소유자와 그룹 변경
chgrp group file그룹만 변경
umask기본 생성 권한 마스크 확인
chmod 1777 /sharedsticky bit 공유 디렉터리 예

6. 헷갈리는 비교표

구분핵심암기 포인트
chmod권한 변경rwx 숫자/기호
chown소유자 변경user:group 가능
chgrp그룹 변경그룹만 바꿀 때
umask기본 권한에서 제거할 권한생성 후 권한이 아님

7. 시험 함정 포인트

  • chmod 755는 rwxr-xr-x입니다.
  • chmod 644는 rw-r--r--입니다.
  • sticky bit는 실행 파일 소유자 권한 실행이 아니라 공유 디렉터리 삭제 제한입니다.
  • setuid와 setgid 숫자를 혼동하지 마세요.

8. 자주 나오는 선지 표현

  • chmod 640의 권한으로 알맞은 것은?
  • umask 022일 때 새 디렉터리 권한은?
  • sticky bit 설명으로 알맞은 것은?

9. 예상문제

1. chmod 640의 의미는?

  1. ① rw-r-----
  2. ② rwxr-----
  3. ③ rw-rw-r--
  4. ④ r--rw----
정답: 1. 6=rw-, 4=r--, 0=---입니다.

2. 앞자리 1 특수 권한은?

  1. ① setuid
  2. ② setgid
  3. ③ sticky bit
  4. ④ execute
정답: 3. 1은 sticky bit입니다.

10. 최종 암기 요약

r=4 w=2 x=1
755 = rwxr-xr-x
644 = rw-r--r--
umask 022 → 파일 644, 디렉터리 755
4=setuid, 2=setgid, 1=sticky
