3. 권한·소유권·특수권한

rwx 권한, 8진수 계산, chmod/chown/chgrp, umask, setuid, setgid, sticky bit를 “문제 풀이용”으로 자세히 정리합니다.

학습 순서 3

1. 단원 개요

권한은 리눅스 보안의 가장 기본 단위입니다. 파일 하나마다 “소유자(user), 그룹(group), 기타 사용자(other)”에게 읽기(read), 쓰기(write), 실행(execute) 권한을 따로 부여합니다.

시험에서는 -rwxr-xr-- 같은 문자 표기와 754 같은 숫자 표기를 서로 바꾸는 문제, chmod와 chown의 역할을 구분하는 문제가 매우 자주 나옵니다.

관련 CBT 풀기 다음: 4. 프로세스

2. 기말고사에서 중요한 이유

  • 수업 실습에서 SSH 접속, 공유 디렉터리, 사용자 그룹 설정과 함께 권한 문제가 연결됩니다.
  • 리눅스마스터 유형에서는 숫자 권한, 특수 권한, umask가 거의 단골로 출제됩니다.
  • 권한을 잘못 주면 서비스 설정 파일을 수정하지 못하거나, 반대로 보안상 위험한 파일이 될 수 있습니다.
  • “명령어를 외운다”보다 “누가 무엇을 할 수 있는가”를 해석하는 능력이 중요합니다.

3. 리눅스마스터 2급 2차 기출 연계 포인트

  • r=4, w=2, x=1을 더해 7, 6, 5, 4, 0을 계산합니다.
  • chmod 755 file은 소유자 rwx, 그룹 r-x, 기타 r-x입니다.
  • chown user:group file은 소유자와 소유 그룹을 함께 변경할 수 있습니다.
  • umask 022일 때 일반 파일 기본 권한은 보통 644, 디렉터리는 755로 이해합니다.
  • setuid, setgid, sticky bit는 4xxx, 2xxx, 1xxx 또는 문자 표기의 s, t로 표현됩니다.

4. 핵심 개념 설명

권한 표기는 3명씩 끊어 읽는다

-rwxr-xr--에서 첫 글자는 파일 종류이고, 그 뒤 9글자를 3개씩 끊습니다. rwx는 소유자, r-x는 그룹, r--는 기타 사용자 권한입니다. 디렉터리는 첫 글자가 d로 보입니다.

파일과 디렉터리에서 x의 의미가 다르다

파일의 실행 권한 x는 프로그램이나 스크립트를 실행할 수 있음을 뜻합니다. 디렉터리의 x는 그 디렉터리 안으로 들어가거나 경로 탐색을 할 수 있음을 뜻합니다. 디렉터리에 읽기 권한만 있고 실행 권한이 없으면 목록을 보더라도 내부 파일 접근이 제한될 수 있습니다.

umask는 기본 권한에서 빼는 값

새 파일은 보통 666에서, 새 디렉터리는 777에서 umask 값을 제거한다고 이해하면 쉽습니다. 예를 들어 umask가 022이면 파일은 644, 디렉터리는 755가 됩니다. 일반 파일에 기본 실행 권한이 붙지 않는다는 점이 중요합니다.

특수 권한은 “권한 판정 규칙을 조금 바꾸는 장치”

setuid는 실행 파일이 파일 소유자의 권한으로 실행될 수 있게 하고, setgid는 실행 파일에서는 그룹 권한, 디렉터리에서는 새 파일의 그룹 상속과 관련됩니다. sticky bit는 공유 디렉터리에서 남의 파일 삭제를 제한할 때 사용합니다.

7. 시험 함정 포인트

  • chmod는 권한 변경, chown은 소유자 변경, chgrp는 그룹 변경입니다.
  • 일반 파일의 기본 권한 계산은 777이 아니라 666에서 시작한다고 보는 것이 안전합니다.
  • 디렉터리의 x 권한은 실행 파일 실행이 아니라 경로 접근 권한입니다.
  • sticky bit는 “모든 사용자가 삭제 가능”이 아니라 오히려 남의 파일 삭제를 제한합니다.
  • setuid는 디렉터리에 설정해도 일반적인 실행 파일 setuid 의미와 다르게 이해해야 합니다. 시험에서는 보통 실행 파일 기준으로 묻습니다.

8. 자주 나오는 선지 표현

  • “755를 문자 권한으로 표현하면?” → rwxr-xr-x
  • “소유자와 그룹을 동시에 변경한다” → chown user:group file
  • “umask 022의 기본 디렉터리 권한” → 755
  • “실행 시 파일 소유자 권한으로 동작” → setuid
  • “공유 디렉터리에서 자기 파일만 삭제” → sticky bit

9. 예상문제

1. chmod 640 report.txt의 의미로 옳은 것은?

  1. 소유자 rw-, 그룹 r--, 기타 ---
  2. 소유자 rwx, 그룹 r--, 기타 ---
  3. 소유자 r--, 그룹 rw-, 기타 ---
  4. 소유자 rw-, 그룹 rw-, 기타 r--
정답: 1. 6은 rw-, 4는 r--, 0은 ---입니다.

2. /tmp 같은 공유 디렉터리에 sticky bit를 설정하는 주된 이유는?

  1. 모든 파일을 root 소유로 만들기 위해
  2. 남의 파일 삭제를 제한하기 위해
  3. 자동 압축을 수행하기 위해
  4. 모든 사용자의 읽기 권한을 제거하기 위해
정답: 2. sticky bit는 공유 디렉터리에서 삭제 권한을 제한합니다.

3. umask가 022일 때 일반 파일과 디렉터리의 기본 권한으로 적절한 것은?

  1. 파일 755, 디렉터리 644
  2. 파일 644, 디렉터리 755
  3. 파일 666, 디렉터리 777
  4. 파일 600, 디렉터리 700
정답: 2. 파일은 666-022=644, 디렉터리는 777-022=755로 이해합니다.

10. 최종 암기 요약

  • r=4, w=2, x=1. 세 자리 숫자는 user/group/other 순서.
  • chmod 권한, chown 소유자, chgrp 그룹.
  • umask 022 → 파일 644, 디렉터리 755.
  • setuid=4, setgid=2, sticky=1.
  • 디렉터리 x는 “들어갈 수 있는 권한”으로 이해한다.
