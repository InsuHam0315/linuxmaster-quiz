5. 셸·환경변수·스크립트·vi

명령어가 실행되는 원리, 환경변수, 리다이렉션, 파이프, 셸 스크립트 실행 방식, vi 편집기의 모드와 핵심 명령을 한 번에 정리합니다.

학습 순서 5

1. 단원 개요

셸은 사용자가 입력한 명령을 운영체제에 전달하는 통역사입니다. 터미널에서 ls, cd, chmod를 입력하면 셸이 PATH에서 실행 파일을 찾고, 필요한 입력·출력 방향을 정한 뒤 프로세스를 실행합니다. vi는 터미널 환경에서 설정 파일을 수정할 때 가장 자주 만나는 편집기입니다.

이 단원은 “명령어 자체”보다 명령이 어떤 환경에서 실행되고, 출력이 어디로 가며, 편집기 안에서 지금 어떤 모드인지를 묻는 문제가 많습니다.

관련 CBT 풀기 다음: 6. 패키지·백업

2. 기말고사에서 중요한 이유

  • 셸 환경을 모르면 source와 bash script.sh의 차이를 설명할 수 없습니다.
  • 환경변수와 PATH는 “명령어가 실행되지 않는 이유”를 찾는 문제로 자주 연결됩니다.
  • 리다이렉션과 파이프는 로그 저장, 에러 저장, 명령 조합 문제에서 단골입니다.
  • vi는 서버에서 설정 파일을 열었을 때 저장, 종료, 검색, 치환을 바로 해야 하므로 실습형 문제로 자주 출제됩니다.

3. 리눅스마스터 2급 2차 기출 연계 포인트

  • export는 셸 변수를 자식 프로세스가 볼 수 있는 환경변수로 내보내는 명령입니다.
  • PATH는 명령어 검색 경로이며, 현재 디렉터리 실행은 보통 ./script.sh처럼 명시합니다.
  • >, >>, 2>, 2>&1의 차이가 선지로 자주 나옵니다.
  • vi의 i, a, o, Esc, :wq, :q!, dd, yy, p, /문자열을 구분해야 합니다.

4. 핵심 개념 설명

셸 변수와 환경변수

NAME=value로 만든 값은 현재 셸 안에서만 보이는 셸 변수입니다. 이 값을 실행되는 프로그램도 보게 하려면 export NAME 또는 export NAME=value를 사용합니다. 시험에서는 “현재 셸에서만 유효한가, 자식 프로세스에도 전달되는가”를 물어봅니다.

PATH가 중요한 이유

셸은 명령어를 입력받으면 PATH에 등록된 디렉터리를 순서대로 검색합니다. 현재 디렉터리가 PATH에 없다면 파일이 있어도 이름만 입력해서 실행되지 않습니다. 그래서 실습에서는 ./backup.sh처럼 현재 디렉터리의 파일임을 명확히 표시합니다.

리다이렉션과 파이프

표준입력은 0, 표준출력은 1, 표준에러는 2입니다. >는 출력 덮어쓰기, >>는 출력 이어쓰기, 2>는 에러만 파일로 보내기입니다. |는 앞 명령의 표준출력을 뒤 명령의 표준입력으로 넘깁니다.

vi는 모드가 전부다

vi는 명령 모드와 입력 모드가 분리됩니다. 글자를 입력하려면 i, a, o 등으로 입력 모드에 들어가고, 명령을 내리려면 Esc로 명령 모드로 돌아와야 합니다. 저장과 종료 명령은 :로 시작하는 ex 명령입니다.

7. 시험 함정 포인트

  • VAR=value만 하면 자식 프로세스가 항상 볼 수 있는 것이 아닙니다. export가 필요합니다.
  • source는 현재 셸, bash script.sh는 새 셸입니다. 환경 설정 반영 문제에서 자주 나옵니다.
  • >는 덮어쓰기, >>는 이어쓰기입니다. 로그 파일 문제에서 둘을 바꿔 내는 선지가 많습니다.
  • 2>&1은 “표준에러를 표준출력과 같은 곳으로” 보내는 표현입니다. 순서가 중요합니다.
  • vi에서 글자가 입력되지 않는다고 당황하면 안 됩니다. 먼저 i로 입력 모드인지, Esc로 명령 모드인지 구분해야 합니다.

8. 자주 나오는 선지 표현

  • “셸 변수를 환경변수로 등록하는 명령은?” → export
  • “명령어 검색 경로를 저장하는 환경변수는?” → PATH
  • “표준에러까지 로그 파일에 저장하는 표현은?” → > log 2>&1
  • “vi에서 저장하지 않고 종료하는 명령은?” → :q!
  • “현재 줄을 복사하고 붙여넣는 vi 명령은?” → yy, p

9. 예상문제

1. 셸 변수 MODE=test를 자식 프로세스도 볼 수 있게 하려면?

  1. alias MODE
  2. export MODE
  3. chmod MODE
  4. source MODE
정답: 2. export는 셸 변수를 환경변수로 내보냅니다.

2. 기존 로그 뒤에 명령 결과를 추가하려면?

  1. cmd > app.log
  2. cmd >> app.log
  3. cmd 2< app.log
  4. cmd | app.log
정답: 2. >>는 파일 끝에 이어 씁니다.

3. vi에서 파일 전체의 old를 new로 치환하는 명령은?

  1. :s/old/new
  2. :%s/old/new/g
  3. :/old/new/all
  4. :%r/old/new
정답: 2. %는 전체 범위, g는 각 줄에서 모든 항목을 의미합니다.

10. 최종 암기 요약

  • PATH는 “명령어 찾는 경로”, export는 “자식 프로세스에게도 보이게 하기”.
  • > 덮어쓰기, >> 이어쓰기, 2> 에러, | 출력 연결.
  • source는 현재 셸, bash script.sh는 새 셸.
  • vi는 입력 모드와 명령 모드를 구분한다. 저장/종료는 Esc 후 :wq 또는 :q!.
