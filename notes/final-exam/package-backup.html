6. 패키지·압축·백업

APT/DPKG, RPM/DNF, 저장소와 GPG, tar와 압축 도구, 백업·복원 명령을 시험에 나오는 방식으로 정리합니다.

학습 순서 6

1. 단원 개요

패키지 관리는 프로그램을 설치, 삭제, 업그레이드하는 체계입니다. 리눅스에서는 배포판 계열에 따라 Debian/Ubuntu 계열의 APT/DPKG와 Red Hat 계열의 RPM/DNF 또는 YUM을 구분합니다.

압축과 백업은 여러 파일을 하나로 묶고, 크기를 줄이고, 필요할 때 복원하는 기술입니다. 시험에서는 tar 옵션 c, x, v, f, z, j, J를 구분하는 문제가 많습니다.

관련 CBT 풀기 다음: 7. 네트워크

2. 기말고사에서 중요한 이유

  • 노션 11주차 자료에서 repository, APT, GPG, MySQL 설치, systemctl 확인 흐름이 강조되었습니다.
  • 교수님 PDF 9장 범위에서 deb/rpm 패키지, apt/dpkg, rpm, tar/zip/gzip 계열이 다뤄집니다.
  • 패키지 명령은 배포판 계열별로 답이 달라서 “Ubuntu인가 Red Hat 계열인가”를 먼저 읽어야 합니다.
  • 백업은 단순 압축이 아니라 “묶기, 압축, 저장 위치, 복원 가능성”을 함께 봐야 합니다.

3. 리눅스마스터 2급 2차 기출 연계 포인트

  • apt update는 저장소 패키지 목록 갱신, apt upgrade는 설치된 패키지 업그레이드입니다.
  • dpkg -i package.deb는 로컬 deb 파일 설치, dpkg -l은 설치 패키지 목록 확인입니다.
  • rpm -qa는 설치된 rpm 패키지 전체 조회, rpm -e는 제거입니다.
  • tar -cvf는 새 아카이브 생성, tar -xvf는 해제, tar -tvf는 목록 확인입니다.
  • gzip, bzip2, xz는 압축 방식이고, tar는 원래 여러 파일을 묶는 아카이브 도구입니다.

4. 핵심 개념 설명

고수준 도구와 저수준 도구

APT와 DNF/YUM은 저장소에서 패키지를 찾고 의존성까지 해결해 주는 고수준 도구입니다. DPKG와 RPM은 개별 패키지 파일을 직접 다루는 저수준 도구입니다. 그래서 초보자는 보통 apt install이나 dnf install을 먼저 떠올리면 됩니다.

저장소와 GPG

저장소(repository)는 패키지 목록과 패키지 파일을 제공하는 서버입니다. GPG 키는 패키지가 신뢰할 수 있는 출처에서 왔는지 확인하는 데 사용됩니다. 수업 자료의 MySQL 저장소 추가 실습은 “저장소 등록 → 목록 갱신 → 설치 → 서비스 확인” 흐름으로 이해하면 됩니다.

tar는 압축이 아니라 묶기에서 출발한다

tar는 여러 파일과 디렉터리를 하나의 아카이브 파일로 묶는 도구입니다. 여기에 gzip, bzip2, xz 같은 압축을 함께 적용할 수 있습니다. 그래서 .tar는 묶기만, .tar.gz는 tar로 묶고 gzip으로 압축한 형태입니다.

백업은 복원까지 생각해야 한다

백업 파일을 만드는 것만으로 끝나지 않습니다. 실제 시험과 실무에서는 “어디에 저장했는지”, “권한과 소유자가 유지되는지”, “복원 명령을 알고 있는지”가 중요합니다.

7. 시험 함정 포인트

  • apt update는 패키지 목록 갱신이지 설치된 패키지를 업그레이드하는 명령이 아닙니다.
  • apt remove와 apt purge를 구분합니다. purge는 설정 파일 제거까지 포함합니다.
  • dpkg는 deb, rpm은 rpm입니다. 확장자와 도구를 섞은 선지를 조심합니다.
  • tar -cvf와 tar -xvf에서 c와 x 하나 차이가 생성/해제를 바꿉니다.
  • xz는 tar -J 대문자 J입니다. j는 bzip2입니다.
  • 백업 파일을 만들 때 절대 경로를 그대로 넣으면 복원 위치가 의도와 달라질 수 있습니다. -C 옵션 의미를 알아두세요.

8. 자주 나오는 선지 표현

  • “저장소 패키지 목록을 갱신한다” → apt update
  • “설치된 RPM 패키지를 모두 조회한다” → rpm -qa
  • “로컬 deb 패키지를 설치한다” → dpkg -i
  • “tar 아카이브를 새로 생성한다” → tar -cvf
  • “gzip 압축 tar를 만든다” → tar -czvf
  • “xz 압축 tar를 만든다” → tar -cJvf

9. 예상문제

1. Ubuntu에서 저장소 패키지 목록 정보를 갱신하는 명령은?

  1. apt update
  2. apt remove
  3. dpkg -e
  4. rpm -qa
정답: 1. apt update는 저장소 인덱스를 갱신합니다.

2. 여러 파일을 gzip 압축 tar 파일로 묶는 명령은?

  1. tar -xvf backup.tar.gz dir
  2. tar -czvf backup.tar.gz dir
  3. gzip -cvf dir.tar
  4. rpm -czvf backup.tar.gz dir
정답: 2. c는 생성, z는 gzip, v는 과정 출력, f는 파일명 지정입니다.

3. RPM 계열에서 설치된 모든 패키지를 조회하는 명령은?

  1. rpm -qa
  2. rpm -e
  3. dpkg -l
  4. apt list
정답: 1. q는 query, a는 all입니다.

10. 최종 암기 요약

  • Ubuntu: apt/dpkg/.deb, Red Hat: dnf 또는 yum/rpm/.rpm.
  • apt update는 목록 갱신, apt upgrade는 업그레이드.
  • tar -cvf 생성, tar -xvf 해제, tar -tvf 목록.
  • gzip=z, bzip2=j, xz=J.
  • 백업은 생성뿐 아니라 복원 위치와 권한 유지까지 생각한다.
