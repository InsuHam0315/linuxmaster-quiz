9. 리눅스마스터 2급 2차 연계 포인트

기말 범위와 겹치는 리눅스마스터 2급 2차 공개 기출·복원 유형을 원문 복사가 아니라 개념, 선지 함정, 출제 포인트 중심으로 재구성해 정리합니다.

학습 순서 9

1. 단원 개요

이 페이지는 자격증 전체 대비가 아니라 운영체제 기말고사를 잘 보기 위한 보조 노트입니다. 리눅스마스터 2급 2차 문제에서 자주 나오는 리눅스 명령어, 권한, 프로세스, 패키지, 네트워크, systemd 유형 중 기말 범위와 겹치는 것만 선별했습니다.

중요한 점은 원문 문제를 외우는 것이 아니라, 어떤 개념을 어떤 함정 선지로 물어보는지를 익히는 것입니다. CBT 문제도 같은 원칙으로 새 문장으로 재구성되어 있습니다.

기출 연계 CBT 풀기 1번 단원부터 복습

2. 기말고사에서 중요한 이유

  • 기말고사는 수업 실습 중심이지만, 명령어와 옵션을 묻는 방식은 CBT형 자격증 문제와 매우 비슷합니다.
  • 리눅스마스터 문제는 선지가 짧고 비슷한 명령을 섞어 내므로, 시험장에서 빠르게 오답을 제거하는 연습에 좋습니다.
  • 파일/권한/프로세스/패키지/네트워크/systemd는 운영체제 수업과 리눅스마스터가 가장 많이 겹치는 부분입니다.
  • 반대로 X 윈도우, 리눅스 활용 분야, 사무용 응용 등 기말 범위와 직접 관련이 낮은 내용은 이 노트에서 우선순위를 낮췄습니다.

4. 핵심 개념 설명

기출 유형은 “정확한 용어 구분”을 노린다

리눅스마스터형 문제는 긴 설명보다 짧은 선지에서 비슷한 용어를 바꿔 내는 경우가 많습니다. 예를 들어 df와 du, start와 enable, apt update와 apt upgrade는 모두 실제 기능이 다르지만 초보자는 이름만 보고 헷갈리기 쉽습니다.

명령어는 “목적 → 옵션 → 대상”으로 외운다

명령어를 통째로 암기하기보다 먼저 목적을 잡습니다. “사용량 확인”이면 df/du, “권한 변경”이면 chmod, “소유자 변경”이면 chown, “서비스 자동 시작”이면 systemctl enable처럼 목적에서 명령을 떠올려야 선지 제거가 빨라집니다.

기말 범위와 직접 관련 없는 내용은 과감히 뒤로 미룬다

리눅스마스터 전체 범위에는 X 윈도우, 응용 분야, 라이선스, 리눅스 역사 등도 포함될 수 있습니다. 그러나 이번 CBT의 목적은 운영체제 기말고사이므로 수업 자료와 노션 범위에서 직접 다룬 실습 명령어와 시스템 관리 영역을 우선합니다.

7. 시험 함정 포인트

  • 명령어 이름이 비슷하면 목적어를 먼저 봅니다. 예: df는 파일시스템, du는 디렉터리.
  • 옵션 한 글자가 의미를 바꿉니다. tar -c와 tar -x, rm -i와 rm -f를 특히 조심합니다.
  • 서비스 문제는 “현재 실행”인지 “부팅 시 자동 실행”인지 질문을 먼저 표시합니다.
  • 권한 문제는 숫자를 바로 답하지 말고 user/group/other 세 칸으로 나누어 적습니다.
  • 네트워크 포트 문제는 서비스 이름과 보안 특징을 함께 묶어 외우면 덜 헷갈립니다.
  • 기말 범위 밖의 리눅스마스터 문제는 보조 학습으로만 보고, 수업 PDF와 노션 범위를 우선합니다.

8. 자주 나오는 선지 표현

  • “같은 inode를 공유한다” → 하드 링크
  • “기존 보조 그룹을 유지하면서 추가한다” → usermod -aG
  • “파일 소유자의 권한으로 실행된다” → setuid
  • “프로세스가 잡을 수 없는 강제 종료” → SIGKILL
  • “표준에러를 표준출력과 같은 대상으로 보낸다” → 2>&1
  • “설치 패키지 목록 전체 조회” → Debian은 dpkg -l, RPM은 rpm -qa
  • “암호화 원격 접속” → SSH
  • “부팅 시 자동 시작 등록” → systemctl enable

9. 예상문제

1. 다음 중 “현재 실행”이 아니라 “다음 부팅 때 자동 시작 등록”을 의미하는 명령은?

  1. systemctl start ssh
  2. systemctl enable ssh
  3. systemctl status ssh
  4. systemctl restart ssh
정답: 2. enable은 자동 시작 등록이며, start와 구분해야 합니다.

2. 리눅스마스터형 문제에서 df와 du를 구분하는 설명으로 가장 적절한 것은?

  1. df는 디렉터리 사용량, du는 사용자 목록이다.
  2. df는 파일시스템 사용량, du는 파일/디렉터리 사용량이다.
  3. df는 파일 삭제, du는 파일 복사이다.
  4. df는 네트워크 포트, du는 DNS 질의이다.
정답: 2. df는 파일시스템 단위, du는 특정 파일/디렉터리 단위입니다.

3. Ubuntu 계열과 RPM 계열 도구 연결로 옳은 것은?

  1. Ubuntu: rpm, RPM 계열: dpkg
  2. Ubuntu: apt/dpkg, RPM 계열: dnf/rpm
  3. Ubuntu: tar, RPM 계열: chmod
  4. Ubuntu: systemctl, RPM 계열: vi만 사용
정답: 2. Debian/Ubuntu 계열은 apt/dpkg, Red Hat 계열은 dnf 또는 yum/rpm을 사용합니다.

10. 최종 암기 요약

  • 이번 CBT의 목적은 자격증 전체가 아니라 운영체제 기말고사 대비다.
  • 리눅스마스터형 문제는 “비슷한 명령어 구분”과 “옵션 한 글자”가 핵심이다.
  • 기말 우선순위: 수업 노션 → 교수님 PDF → 실습 명령어 → 리눅스마스터 연계 유형.
  • 모르는 문제가 나오면 카테고리를 먼저 판단하고, 목적에 맞는 명령어를 떠올린다.
