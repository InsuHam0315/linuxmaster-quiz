부팅 과정·systemd·서비스 관리

BIOS/UEFI, MBR, GRUB, 커널, systemd target, systemctl 서비스 관리를 정리합니다.

부팅

1. 단원 개요

부팅은 펌웨어에서 시작해 부트로더, 커널, systemd 초기화로 이어집니다. 서비스 관리는 systemctl로 수행합니다.

관련 CBT 풀기 리눅스마스터 연계 보기

2. 기말고사에서 중요한 이유

기말에서는 GRUB 역할, /boot/vmlinuz, runlevel과 target 대응, systemctl start/enable/status 차이가 중요합니다.

3. 리눅스마스터 2급 2차 기출 연계 포인트

  • GRUB와 LILO
  • BIOS/UEFI → 부트로더 → 커널 → systemd
  • multi-user.target과 graphical.target
  • systemctl get-default/set-default
  • 서비스 start/enable/status/restart

4. 핵심 개념 설명

  • 부트로더는 커널을 메모리에 적재합니다.
  • 커널 이미지는 보통 /boot 아래에 있습니다.
  • systemd는 현대 리눅스에서 초기화와 서비스 관리를 담당합니다.
  • target은 과거 runlevel과 비슷하게 시스템 상태를 묶어 표현합니다.

5. 주요 명령어

명령어핵심 의미
systemctl get-default기본 target 확인
systemctl set-default multi-user.target기본 target 변경
systemctl isolate graphical.target현재 target 전환
systemctl status ssh서비스 상태 확인
systemctl start ssh서비스 즉시 시작
systemctl enable ssh부팅 자동 시작 등록
journalctl -u ssh서비스 로그 확인

6. 헷갈리는 비교표

구분핵심암기 포인트
multi-user.target텍스트 기반 다중 사용자전통적 runlevel 3에 대응
graphical.target그래픽 환경 포함전통적 runlevel 5에 대응
rescue.target복구용 단일 사용자 성격문제 해결
poweroff.target전원 종료시스템 종료

7. 시험 함정 포인트

  • start는 즉시 시작, enable은 자동 시작 등록입니다.
  • get-default는 확인, set-default는 변경입니다.
  • graphical.target은 GUI 포함입니다.
  • 부트로더는 패키지 관리자가 아닙니다.

8. 자주 나오는 선지 표현

  • 현재 기본 target 확인 명령은?
  • 부팅 시 자동 시작 등록 명령은?
  • 그래픽 환경 target은?

9. 예상문제

1. 기본 target 확인 명령은?

  1. ① systemctl get-default
  2. ② systemctl status-default
  3. ③ systemctl list-default
  4. ④ systemctl boot-default
정답: 1. get-default가 기본 target 확인입니다.

2. 서비스를 부팅 시 자동 시작하게 하는 명령은?

  1. ① systemctl start
  2. ② systemctl enable
  3. ③ systemctl status
  4. ④ systemctl stop
정답: 2. enable은 자동 시작 등록입니다.

10. 최종 암기 요약

BIOS/UEFI → GRUB → Kernel → systemd
/boot = 커널/부트 파일
multi-user = 텍스트 다중 사용자
graphical = GUI
start 즉시, enable 자동
