8. 부팅 과정·systemd·서비스 관리

전원 입력부터 로그인 화면까지의 부팅 흐름, GRUB, 커널, initramfs, systemd target, systemctl 서비스 명령을 정리합니다.

학습 순서 8

1. 단원 개요

부팅은 컴퓨터 전원을 켠 뒤 운영체제가 사용할 준비를 마치는 과정입니다. 하드웨어 초기화, 부트로더 실행, 커널 적재, 루트 파일시스템 연결, init/systemd 실행, 서비스 시작이 순서대로 이어집니다.

systemd는 현대 리눅스에서 서비스와 부팅 타겟을 관리하는 핵심 시스템입니다. systemctl start, enable, status의 차이를 모르면 서비스 관리 문제에서 자주 틀립니다.

관련 CBT 풀기 다음: 9. 연계 포인트

2. 기말고사에서 중요한 이유

  • 교수님 PDF 8장 범위에서 부팅, 종료, BIOS/MBR, 부트로더, 커널, GRUB/LILO, runlevel/systemd가 다뤄집니다.
  • 노션 자료의 MySQL, SSH, FTP, MariaDB 실습은 설치 후 systemctl로 서비스를 확인하고 시작하는 흐름과 연결됩니다.
  • 부팅 문제는 순서 문제가 많고, systemd 문제는 명령어 의미 차이 문제가 많습니다.
  • 서비스가 “지금 실행 중인지”와 “다음 부팅 때 자동 시작되는지”는 서로 다른 질문입니다.

3. 리눅스마스터 2급 2차 기출 연계 포인트

  • 전통적인 부팅 흐름은 BIOS/UEFI → MBR/GPT/부트로더 → 커널 → init/systemd → 서비스 순서로 이해합니다.
  • GRUB은 리눅스에서 대표적인 부트로더이며 커널과 initramfs를 메모리에 올리는 역할을 합니다.
  • systemd target은 과거 runlevel과 대응되는 개념으로 자주 출제됩니다.
  • systemctl start는 즉시 시작, enable은 부팅 시 자동 시작 등록입니다.
  • multi-user.target은 텍스트 기반 다중 사용자 환경, graphical.target은 GUI 환경에 가깝습니다.

4. 핵심 개념 설명

부팅 순서

전원이 켜지면 펌웨어(BIOS/UEFI)가 하드웨어를 초기화하고 부팅 가능한 장치를 찾습니다. 그 다음 부트로더가 실행되어 커널을 메모리에 올립니다. 커널은 하드웨어와 파일시스템을 인식하고, 초기 루트 환경(initramfs)을 이용해 실제 루트 파일시스템을 마운트한 뒤 첫 사용자 공간 프로세스인 systemd를 실행합니다.

부트로더와 GRUB

부트로더는 운영체제 커널을 찾아 실행하는 프로그램입니다. GRUB은 여러 커널이나 운영체제 중 무엇으로 부팅할지 선택할 수 있게 해 줍니다. 시험에서는 GRUB을 “커널 그 자체”로 설명하는 선지를 조심해야 합니다.

runlevel과 target

과거 SysV init에서는 runlevel로 시스템 상태를 구분했습니다. systemd에서는 target이 유사한 역할을 합니다. 예를 들어 multi-user.target은 네트워크와 다중 사용자 텍스트 환경, graphical.target은 GUI까지 포함한 환경입니다.

서비스 관리의 두 축

서비스 관리는 “지금 실행”과 “자동 시작 등록”을 분리해서 봐야 합니다. systemctl start nginx는 지금 시작하지만 다음 부팅 때 자동 실행을 보장하지 않습니다. systemctl enable nginx는 다음 부팅 때 자동 시작되도록 등록하지만, 이미 실행 중이라는 뜻은 아닙니다.

7. 시험 함정 포인트

  • enable은 지금 즉시 서비스를 시작하는 명령이 아닙니다. 자동 시작 등록입니다.
  • start는 지금 시작하지만 다음 부팅 자동 시작을 보장하지 않습니다.
  • GRUB은 커널이 아니라 커널을 찾아 실행하는 부트로더입니다.
  • runlevel 0은 종료, 6은 재부팅입니다. 둘을 바꾸는 선지를 조심하세요.
  • multi-user.target과 graphical.target의 차이는 GUI 포함 여부로 기억하면 쉽습니다.
  • 서비스 상태는 status, 로그는 journalctl로 확인합니다.

8. 자주 나오는 선지 표현

  • “부팅 시 자동 시작되도록 등록한다” → systemctl enable
  • “현재 서비스를 즉시 시작한다” → systemctl start
  • “기본 부팅 target 확인” → systemctl get-default
  • “systemd 서비스 로그 확인” → journalctl -u 서비스명
  • “텍스트 기반 다중 사용자 target” → multi-user.target
  • “GUI 포함 target” → graphical.target

9. 예상문제

1. SSH 서비스를 다음 부팅 때 자동으로 시작되도록 등록하는 명령은?

  1. systemctl start ssh
  2. systemctl enable ssh
  3. systemctl status ssh
  4. systemctl stop ssh
정답: 2. enable은 자동 시작 등록입니다.

2. systemd에서 GUI가 포함된 기본 target에 가까운 것은?

  1. poweroff.target
  2. rescue.target
  3. multi-user.target
  4. graphical.target
정답: 4. graphical.target은 GUI 환경까지 포함합니다.

3. 부팅 과정에서 커널을 찾아 메모리에 올리는 역할을 하는 대표 구성요소는?

  1. GRUB
  2. passwd
  3. cron
  4. chmod
정답: 1. GRUB은 대표적인 리눅스 부트로더입니다.

10. 최종 암기 요약

  • 부팅 흐름: BIOS/UEFI → 부트로더 → 커널 → initramfs → systemd → 서비스.
  • GRUB은 부트로더, 커널 자체가 아니다.
  • start는 지금 시작, enable은 자동 시작 등록.
  • runlevel 0 종료, 1 복구, 3 multi-user, 5 graphical, 6 재부팅.
  • 서비스 상태는 systemctl status, 로그는 journalctl.
