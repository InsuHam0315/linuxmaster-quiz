7. 네트워크 기초·서비스

IP, MAC, TCP/UDP, 포트, DNS, 라우팅, ping/traceroute/ss, SSH/Telnet/FTP/HTTP/DB 서비스를 기말 범위 중심으로 정리합니다.

학습 순서 7

1. 단원 개요

네트워크는 컴퓨터끼리 데이터를 주고받기 위한 규칙과 장치의 모음입니다. 운영체제 관점에서는 “내 컴퓨터의 주소가 무엇인지”, “어느 서비스가 어느 포트에서 기다리는지”, “문제가 생겼을 때 어디까지 통신되는지”를 확인할 수 있어야 합니다.

수업 자료의 네트워크 장에서는 MAC 주소, IP 주소, 라우팅, DNS, ping, traceroute, netstat, ifconfig 같은 기본 도구가 다뤄졌고, 노션 자료에서는 SSH, Telnet, FTP, MariaDB 같은 서비스 실습이 연결됩니다.

관련 CBT 풀기 다음: 8. 부팅·systemd

2. 기말고사에서 중요한 이유

  • 서버 접속과 서비스 운영은 대부분 네트워크 포트와 방화벽, 서비스 상태 확인으로 연결됩니다.
  • SSH, Telnet, FTP처럼 비슷해 보이는 원격 접속/전송 서비스의 보안 차이를 구분해야 합니다.
  • 네트워크 문제 해결은 단계적으로 해야 합니다. IP 설정, 게이트웨이, DNS, 서비스 포트, 방화벽 순서로 확인합니다.
  • 리눅스마스터 유형에서는 주요 포트 번호와 진단 명령의 목적을 묻는 문제가 많습니다.

3. 리눅스마스터 2급 2차 기출 연계 포인트

  • SSH는 22번 포트를 사용하며 암호화된 원격 접속입니다. Telnet은 23번이며 평문 전송이라 보안상 취약합니다.
  • FTP는 제어 연결 21번, 데이터 연결 20번 개념이 자주 언급됩니다.
  • HTTP는 80, HTTPS는 443, DNS는 53, SMTP는 25, POP3는 110, IMAP은 143을 기본 포트로 기억합니다.
  • ping은 ICMP를 이용해 도달 가능성을 확인하고, traceroute는 목적지까지 경로를 확인합니다.
  • ss -tuln 또는 netstat -tuln은 열린 포트/리스닝 소켓 확인에 사용됩니다.

4. 핵심 개념 설명

MAC 주소와 IP 주소

MAC 주소는 네트워크 카드에 가까운 하드웨어 주소이고, 같은 네트워크 안에서 프레임을 전달할 때 사용됩니다. IP 주소는 네트워크 계층 주소로, 서로 다른 네트워크 사이에서 목적지를 찾는 데 사용됩니다. 쉽게 말해 MAC은 “근처에서 찾는 이름표”, IP는 “네트워크 전체에서 찾는 주소”에 가깝습니다.

포트는 서비스의 문 번호

서버 한 대에는 여러 서비스가 동시에 실행될 수 있습니다. IP 주소가 서버 위치라면 포트 번호는 그 서버 안에서 어떤 서비스로 들어갈지 정하는 문 번호입니다. 예를 들어 같은 서버라도 22번은 SSH, 80번은 HTTP일 수 있습니다.

TCP와 UDP

TCP는 연결을 맺고 순서와 신뢰성을 챙기는 방식입니다. 웹, SSH, FTP처럼 안정적인 전달이 중요한 서비스에 많이 쓰입니다. UDP는 연결 과정이 단순하고 빠르지만 신뢰성 처리는 응용 프로그램이 담당합니다. DNS 질의나 스트리밍 일부에서 사용됩니다.

DNS와 라우팅

DNS는 도메인 이름을 IP 주소로 바꾸는 시스템입니다. 라우팅은 목적지 네트워크로 가기 위해 다음에 보낼 장치, 즉 게이트웨이를 결정하는 과정입니다. 웹사이트 접속이 안 될 때 IP로는 접속되는데 도메인만 안 되면 DNS 문제를 의심할 수 있습니다.

7. 시험 함정 포인트

  • SSH와 Telnet은 둘 다 원격 접속이지만 SSH는 암호화, Telnet은 평문입니다.
  • ping이 성공해도 특정 포트가 열려 있다는 뜻은 아닙니다. 포트 확인은 ss, netstat, 또는 서비스 접속 테스트가 필요합니다.
  • DNS 문제와 네트워크 단절 문제를 구분합니다. IP로 ping은 되는데 도메인만 안 되면 DNS를 의심합니다.
  • FTP의 20/21 포트, DNS의 53 포트는 자주 뒤섞여 나옵니다.
  • ifconfig와 netstat은 구형 도구지만 시험에는 여전히 자주 보입니다. 현대 명령인 ip, ss와 함께 기억하세요.

8. 자주 나오는 선지 표현

  • “암호화된 원격 접속 서비스” → SSH, 22번 포트
  • “도메인 이름을 IP 주소로 변환” → DNS, 53번 포트
  • “목적지까지 거치는 경로 확인” → traceroute
  • “현재 리스닝 중인 TCP/UDP 포트 확인” → ss -tuln
  • “기본 게이트웨이 확인” → ip route

9. 예상문제

1. SSH 서비스의 기본 포트와 특징으로 옳은 것은?

  1. 21번, 파일 목록 압축
  2. 22번, 암호화 원격 접속
  3. 23번, 암호화 웹 접속
  4. 53번, 메일 전송
정답: 2. SSH는 기본 22번 포트를 사용하며 암호화된 원격 접속 서비스입니다.

2. 목적지까지 패킷이 거치는 경로를 확인하는 명령은?

  1. ping
  2. traceroute
  3. passwd
  4. chmod
정답: 2. traceroute는 중간 경로를 추적합니다.

3. 현재 서버에서 열린 리스닝 포트를 확인하는 명령으로 적절한 것은?

  1. ss -tuln
  2. tar -tuln
  3. useradd -tuln
  4. mkfs -tuln
정답: 1. ss -tuln은 TCP/UDP 리스닝 소켓을 숫자 형식으로 보여 줍니다.

10. 최종 암기 요약

  • IP는 네트워크 주소, MAC은 장치 주소, 포트는 서비스 문 번호.
  • SSH 22, Telnet 23, FTP 21/20, HTTP 80, HTTPS 443, DNS 53, MySQL/MariaDB 3306.
  • ping 도달성, traceroute 경로, dig/nslookup DNS, ss 포트.
  • 문제 해결 순서: IP → 게이트웨이 → DNS → 포트/서비스 → 방화벽.
