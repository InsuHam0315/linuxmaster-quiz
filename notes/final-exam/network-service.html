네트워크

1. 단원 개요

네트워크는 프로토콜 규칙에 따라 시스템이 데이터를 주고받는 구조입니다. 주소, 라우팅, 이름 해석, 서비스 포트를 함께 봐야 합니다.

관련 CBT 풀기 리눅스마스터 연계 보기

2. 기말고사에서 중요한 이유

기말에서는 ip addr, ip route, ping, traceroute, DNS, SSH/Telnet 차이, FTP 포트가 실습형으로 나오기 쉽습니다.

3. 리눅스마스터 2급 2차 기출 연계 포인트

  • ifconfig/ip addr
  • route/ip route
  • ping/traceroute
  • netstat/ss
  • DNS와 주요 포트
  • SSH와 Telnet 보안 차이

4. 핵심 개념 설명

  • MAC 주소는 네트워크 인터페이스의 물리 주소 성격이 강합니다.
  • IP 주소는 네트워크 계층의 논리 주소입니다.
  • 라우팅은 목적지 네트워크로 패킷을 보낼 다음 경로를 결정합니다.
  • DNS는 도메인 이름과 IP 주소를 변환합니다.
  • SSH는 암호화된 원격 접속이고 Telnet은 평문 기반입니다.

명령어핵심 의미

7. 시험 함정 포인트

  • FTP 제어 포트는 21입니다.
  • SSH는 22, Telnet은 23입니다.
  • ping은 경로 전체가 아니라 응답 가능성 확인에 가깝습니다.
  • traceroute는 경로 추적입니다.
  • ss는 netstat 대체로 자주 쓰입니다.

9. 예상문제

1. SSH 기본 포트는?

  1. ① 21
  2. ② 22
  3. ③ 23
  4. ④ 80
정답: 2. SSH는 기본적으로 TCP 22번 포트를 사용합니다.

2. 목적지까지 라우터 경로를 보는 명령은?

  1. ① ping
  2. ② traceroute
  3. ③ passwd
  4. ④ chmod
정답: 2. traceroute는 홉 경로를 추적합니다.
