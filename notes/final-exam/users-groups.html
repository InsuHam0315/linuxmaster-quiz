사용자 및 그룹

사용자 생성, 그룹 추가, UID/GID, /etc/passwd, /etc/shadow, /etc/group을 정리합니다.

계정관리

1. 단원 개요

리눅스는 다중 사용자 운영체제이므로 사용자와 그룹을 기준으로 접근 권한과 자원 사용을 통제합니다.

관련 CBT 풀기 리눅스마스터 연계 보기

2. 기말고사에서 중요한 이유

기말고사에서는 useradd 옵션, usermod -aG, passwd/shadow/group 파일 역할이 자주 함정으로 나옵니다.

3. 리눅스마스터 2급 2차 기출 연계 포인트

  • useradd -m -s 옵션
  • usermod -aG와 -G 차이
  • /etc/passwd와 /etc/shadow 구분
  • UID/GID 의미
  • adduser와 useradd 비교

4. 핵심 개념 설명

  • UID는 시스템이 사용자를 식별하는 번호이고 GID는 그룹 식별 번호입니다.
  • /etc/passwd는 계정 기본 정보를 담고 실제 암호 해시는 보통 /etc/shadow에 저장됩니다.
  • /etc/group은 그룹 이름, GID, 보조 그룹 구성원 정보를 담습니다.
  • 보조 그룹 추가 시 기존 그룹을 보존하려면 usermod -aG를 사용합니다.

5. 주요 명령어

명령어핵심 의미
adduser user1대화형 사용자 생성
useradd -m -s /bin/bash user1홈 디렉터리와 로그인 셸 지정 생성
passwd user1암호 설정/변경
groupadd project그룹 생성
usermod -aG project user1보조 그룹 추가
id user1UID/GID와 그룹 확인
groups user1소속 그룹 확인

6. 헷갈리는 비교표

구분핵심암기 포인트
adduser배포판에서 제공하는 고수준/대화형 명령초보자 실습에 편리
useradd저수준 계정 생성 명령옵션을 직접 지정해야 함
-g기본 그룹 지정/변경하나의 주 그룹
-G보조 그룹 목록 지정-a 없이 쓰면 기존 목록 대체 주의

7. 시험 함정 포인트

  • -aG에서 -a를 빠뜨리면 기존 보조 그룹이 대체될 수 있습니다.
  • /etc/passwd에 암호 해시 전체가 있다고 고르면 틀리기 쉽습니다.
  • 그룹 생성은 useradd가 아니라 groupadd입니다.
  • 로그인 셸 지정은 useradd -s입니다.

8. 자주 나오는 선지 표현

  • 사용자의 홈 디렉터리를 자동 생성하는 옵션은?
  • 기존 보조 그룹을 유지하며 그룹을 추가하는 명령은?
  • /etc/shadow 파일 설명으로 알맞은 것은?

9. 예상문제

1. 홈 디렉터리를 만들고 bash를 로그인 셸로 지정하는 명령은?

  1. ① useradd -m -s /bin/bash user1
  2. ② groupadd -m user1
  3. ③ passwd -s /bin/bash user1
  4. ④ usermod -m /bin/bash user1
정답: 1. -m은 홈 생성, -s는 셸 지정입니다.

2. 암호 해시가 주로 저장되는 파일은?

  1. ① /etc/passwd
  2. ② /etc/shadow
  3. ③ /etc/group
  4. ④ /etc/fstab
정답: 2. 현대 리눅스의 암호 해시는 /etc/shadow에 저장됩니다.

10. 최종 암기 요약

계정 기본 정보 = /etc/passwd
암호 해시 = /etc/shadow
그룹 정보 = /etc/group
홈 생성 = useradd -m
보조 그룹 추가 = usermod -aG
