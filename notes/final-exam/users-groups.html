2. 사용자 및 그룹

리눅스 계정의 구조, UID/GID, 기본 그룹과 보조 그룹, 계정 정보 파일, useradd/adduser/usermod/passwd 명령을 정리합니다.

학습 순서 2

1. 단원 개요

리눅스는 여러 사용자가 동시에 쓰는 운영체제를 전제로 만들어졌습니다. 그래서 파일을 읽거나 명령을 실행할 때 항상 “현재 사용자가 누구인가”, “그 사용자가 어떤 그룹에 속하는가”를 확인합니다.

사용자와 그룹 단원은 권한 단원의 기초입니다. chmod 계산을 잘해도 UID, GID, 소유자, 소유 그룹을 모르면 실제 접근 권한을 판단할 수 없습니다.

2. 기말고사에서 중요한 이유

  • 수업 노션 자료에서 adduser, useradd, groupadd, usermod -aG가 실습 중심으로 다뤄졌습니다.
  • 계정 정보 파일인 /etc/passwd, /etc/shadow, /etc/group의 역할을 구분해야 합니다.
  • 기본 그룹과 보조 그룹의 차이를 알아야 협업 디렉터리나 서비스 계정 문제를 이해할 수 있습니다.
  • 사용자를 삭제하거나 그룹에 추가하는 명령은 옵션 하나 차이로 결과가 달라져 선지 함정이 많습니다.

3. 리눅스마스터 2급 2차 기출 연계 포인트

  • /etc/passwd는 사용자명, UID, GID, 홈 디렉터리, 로그인 셸 같은 기본 정보를 담습니다.
  • /etc/shadow는 암호 해시와 암호 만료 정보를 담으며 일반 사용자에게 제한됩니다.
  • /etc/group은 그룹명, GID, 보조 그룹 구성원 정보를 담습니다.
  • useradd -m -s /bin/bash user1에서 -m은 홈 디렉터리 생성, -s는 로그인 셸 지정입니다.
  • usermod -aG group user는 기존 보조 그룹을 보존하고 새 그룹을 추가합니다. -a 없이 -G만 쓰면 보조 그룹 목록이 대체될 수 있습니다.

4. 핵심 개념 설명

UID와 GID

리눅스 내부는 사용자 이름보다 숫자인 UID를 기준으로 사용자를 구분합니다. root는 일반적으로 UID 0입니다. 그룹도 이름보다 GID라는 숫자를 갖습니다. 파일 목록에서 이름이 보이는 것은 사람이 보기 쉽게 변환된 결과라고 이해하면 됩니다.

기본 그룹과 보조 그룹

사용자는 기본 그룹 하나를 가지고, 필요에 따라 여러 보조 그룹에 들어갈 수 있습니다. 새 파일을 만들 때 기본 그룹이 파일의 그룹 소유자로 지정되는 경우가 많고, 보조 그룹은 프로젝트 디렉터리 접근 권한을 줄 때 자주 사용합니다.

계정 정보 파일 3종 세트

/etc/passwd는 계정의 공개 기본 정보, /etc/shadow는 민감한 암호 정보, /etc/group은 그룹 정보를 담당합니다. 예전에는 passwd에 암호가 들어갔지만 현대 리눅스에서는 보안상 shadow로 분리합니다.

adduser와 useradd

배포판에 따라 adduser는 사용자 친화적인 스크립트 형태로 동작하고, useradd는 비교적 낮은 수준의 계정 생성 명령으로 동작합니다. 시험에서는 보통 useradd 옵션의 의미를 묻습니다.

7. 시험 함정 포인트

  • /etc/passwd에 암호 해시가 저장된다고 단정하는 선지를 조심합니다. 현대 리눅스에서는 주로 /etc/shadow입니다.
  • usermod -G와 usermod -aG의 차이는 매우 자주 나옵니다. “추가”는 -aG입니다.
  • userdel만 하면 홈 디렉터리가 남을 수 있고, userdel -r은 홈까지 삭제합니다.
  • UID 0은 root 권한과 연결됩니다. 이름이 root가 아니어도 UID 0이면 위험합니다.
  • 사용자 이름과 UID, 그룹 이름과 GID를 혼동하지 마세요. 내부 판단은 숫자 ID 기반입니다.

8. 자주 나오는 선지 표현

  • “사용자의 UID와 GID를 확인한다” → id
  • “암호 해시와 만료 정보를 저장한다” → /etc/shadow
  • “기존 보조 그룹을 유지하면서 새 그룹을 추가한다” → usermod -aG
  • “홈 디렉터리를 만들며 사용자를 생성한다” → useradd -m
  • “로그인 셸을 지정한다” → useradd -s 또는 usermod -s

9. 예상문제

1. 기존 사용자 user1을 project 보조 그룹에 추가하면서 기존 보조 그룹을 유지하는 명령은?

  1. usermod -G project user1
  2. usermod -aG project user1
  3. groupadd -a user1 project
  4. passwd -G project user1
정답: 2. -aG는 append + supplementary group으로 기억하면 좋습니다.

2. 암호 해시와 암호 만료 정보가 저장되는 파일은?

  1. /etc/passwd
  2. /etc/shadow
  3. /etc/group
  4. /etc/fstab
정답: 2. /etc/shadow는 민감한 암호 정보를 담습니다.

3. 홈 디렉터리를 만들며 bash 셸을 지정해 user1을 생성하는 명령은?

  1. useradd -m -s /bin/bash user1
  2. useradd -G /bin/bash -m user1
  3. groupadd -m -s /bin/bash user1
  4. passwd -m -s /bin/bash user1
정답: 1. -m은 홈 생성, -s는 로그인 셸 지정입니다.

10. 최종 암기 요약

  • root는 보통 UID 0이다.
  • /etc/passwd는 기본 계정 정보, /etc/shadow는 암호 정보, /etc/group은 그룹 정보.
  • 사용자 생성: useradd -m -s /bin/bash user.
  • 그룹 추가: usermod -aG group user.
  • 권한 단원을 풀기 전 사용자/그룹 소유자를 먼저 확인한다.
