4. 프로세스 및 작업 관리

실행 중인 프로그램을 확인하고, 우선순위를 조절하고, 신호로 종료하고, 백그라운드 작업과 예약 작업을 관리하는 방법을 정리합니다.

학습 순서 4

1. 단원 개요

프로세스는 실행 중인 프로그램입니다. 같은 프로그램을 두 번 실행하면 프로세스도 두 개가 될 수 있고, 각각은 PID라는 고유 번호를 갖습니다. 운영체제는 CPU와 메모리를 여러 프로세스에 나누어 주며, 사용자는 ps, top, kill 같은 명령으로 상태를 확인하고 제어합니다.

작업 관리는 현재 터미널에서 실행한 명령을 포그라운드와 백그라운드로 다루는 기능입니다. 긴 작업을 &로 뒤에서 실행하거나, jobs, fg, bg로 전환하는 문제가 자주 나옵니다.

관련 CBT 풀기 다음: 5. 셸·vi

2. 기말고사에서 중요한 이유

  • 서버 관리에서 “무슨 프로그램이 CPU를 많이 쓰는지”, “어떤 서비스를 종료해야 하는지”를 판단해야 합니다.
  • 리눅스마스터 유형에서는 ps aux, top, kill -9, nice, jobs가 반복 출제됩니다.
  • 프로세스와 서비스는 연결됩니다. 서비스 관리(systemd)를 이해하려면 먼저 프로세스 개념을 알아야 합니다.
  • 예약 작업인 cron/at은 백업, 로그 정리, 자동 실행 문제로 자주 연결됩니다.

3. 리눅스마스터 2급 2차 기출 연계 포인트

  • ps aux는 전체 사용자 프로세스를 자세히 볼 때 자주 쓰는 조합입니다.
  • top은 CPU/메모리 사용량을 실시간으로 확인합니다.
  • SIGTERM은 정상 종료 요청, SIGKILL은 잡을 수 없는 강제 종료 신호입니다.
  • nice 값은 낮을수록 일반적으로 우선순위가 높고, 높을수록 양보하는 쪽입니다.
  • nohup command &는 터미널 종료 후에도 작업을 계속 실행할 때 사용합니다.

4. 핵심 개념 설명

PID와 PPID

PID는 프로세스 번호이고 PPID는 부모 프로세스 번호입니다. 사용자가 터미널에서 명령을 실행하면 셸이 부모가 되고 명령이 자식 프로세스로 실행됩니다. 프로세스 트리를 보면 어떤 프로그램이 무엇을 실행했는지 추적할 수 있습니다.

포그라운드와 백그라운드

포그라운드 작업은 현재 터미널을 점유합니다. 명령 뒤에 &를 붙이면 백그라운드에서 실행되어 터미널을 계속 사용할 수 있습니다. Ctrl+Z는 현재 작업을 일시 중지하고, bg는 백그라운드 재개, fg는 포그라운드 복귀입니다.

신호(signal)

프로세스 제어는 신호를 보내는 방식으로 이루어집니다. kill PID는 기본적으로 SIGTERM을 보내 정상 종료를 요청합니다. kill -9 PID는 SIGKILL로 즉시 종료시키지만 정리 작업을 할 기회를 주지 않으므로 마지막 수단으로 이해합니다.

우선순위와 nice

nice 값은 프로세스가 CPU를 얼마나 “양보”할지를 나타냅니다. 값이 낮을수록 우선순위가 높아지는 방향이고, 값이 높을수록 우선순위가 낮아지는 방향입니다. 일반 사용자는 보통 우선순위를 높이는 데 제한이 있습니다.

7. 시험 함정 포인트

  • kill이라는 이름 때문에 무조건 강제 종료라고 생각하면 안 됩니다. 기본은 SIGTERM입니다.
  • kill -9는 강력하지만 정상 정리 없이 종료하므로 항상 첫 선택은 아닙니다.
  • jobs는 시스템 전체가 아니라 현재 셸의 작업 목록입니다. 전체 프로세스는 ps나 top으로 봅니다.
  • nice 값이 높을수록 좋은 우선순위가 아니라, 더 “친절하게 양보”하는 쪽입니다.
  • nohup만으로 백그라운드 실행이 되는 것이 아니라 보통 &를 붙여 터미널을 돌려받습니다.

8. 자주 나오는 선지 표현

  • “전체 사용자 프로세스를 자세히 확인한다” → ps aux
  • “실시간 CPU/메모리 사용량을 확인한다” → top
  • “잡을 수 없는 강제 종료 신호” → SIGKILL, kill -9
  • “현재 셸의 백그라운드 작업 목록” → jobs
  • “터미널 종료 후에도 명령 실행 유지” → nohup command &
  • “반복 예약 작업” → cron, “1회 예약 작업” → at

9. 예상문제

1. SIGTERM과 SIGKILL의 설명으로 옳은 것은?

  1. SIGTERM은 강제 종료, SIGKILL은 종료 요청이다.
  2. SIGTERM은 정상 종료 요청, SIGKILL은 잡을 수 없는 강제 종료이다.
  3. 둘 다 파일 권한 변경 명령이다.
  4. 둘 다 백업 예약 명령이다.
정답: 2. SIGTERM은 정상 종료 요청이고 SIGKILL은 강제 종료입니다.

2. 현재 셸의 백그라운드 작업 목록을 확인하는 명령은?

  1. jobs
  2. fg
  3. bg
  4. nohup
정답: 1. jobs는 현재 셸이 관리하는 작업 목록을 보여 줍니다.

3. 터미널을 닫아도 장시간 작업이 계속되도록 실행하는 형태로 적절한 것은?

  1. nohup command &
  2. chmod command &
  3. fsck command &
  4. umask command
정답: 1. nohup은 SIGHUP 영향을 피하고, &는 백그라운드 실행을 의미합니다.

10. 최종 암기 요약

  • 프로세스는 실행 중인 프로그램, PID는 프로세스 번호.
  • ps는 스냅샷, top은 실시간.
  • kill 기본은 SIGTERM, kill -9는 SIGKILL.
  • jobs, fg, bg는 현재 셸 작업 제어.
  • cron은 반복 예약, at은 1회 예약.
