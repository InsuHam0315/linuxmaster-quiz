파일시스템 구조, inode, 링크, 디스크 파티션, LVM, mkfs, mount를 기말 실습 기준으로 정리합니다.

파일시스템

1. 단원 개요

운영체제가 파일을 디스크에 저장하고 찾는 방식, 새 디스크를 추가해 파티션과 파일시스템을 만들고 마운트하는 흐름을 다룹니다.

관련 CBT 풀기 리눅스마스터 연계 보기

2. 기말고사에서 중요한 이유

기말고사에서는 lsblk, fdisk, mkfs.ext4, mount, umount, /etc/fstab, LVM 순서를 명령어 기반으로 묻기 쉽습니다.

3. 리눅스마스터 2급 2차 기출 연계 포인트

  • inode와 하드 링크/심볼릭 링크 구분
  • df와 du의 차이
  • mount/umount 명령과 /etc/fstab
  • LVM의 PV → VG → LV 순서
  • tar 백업과 파일시스템 점검 함정

4. 핵심 개념 설명

  • 파일시스템은 파일 이름, 메타데이터, 데이터 블록을 체계적으로 관리하는 구조입니다.
  • inode는 권한, 소유자, 크기, 시간, 데이터 블록 위치 같은 메타데이터를 담습니다.
  • 디렉터리는 파일 이름과 inode 번호를 연결하는 표 역할을 합니다.
  • 마운트는 별도 파일시스템을 기존 디렉터리 트리에 연결하는 작업입니다.
  • LVM은 물리 디스크/파티션을 논리적으로 묶어 유연한 볼륨 관리를 가능하게 합니다.

5. 주요 명령어

명령어핵심 의미
lsblk블록 장치와 파티션 구조 확인
fdisk /dev/sdb디스크 파티션 편집
mkfs.ext4 /dev/sdb1ext4 파일시스템 생성
mount /dev/sdb1 /mnt/data파일시스템 마운트
umount /mnt/data마운트 해제
df -h마운트된 파일시스템 사용량
du -sh DIR디렉터리 실제 사용량 요약
pvcreate, vgcreate, lvcreateLVM 구성

6. 헷갈리는 비교표

구분핵심암기 포인트
하드 링크같은 inode 공유, 같은 파일시스템 내 생성원본 삭제 후에도 링크 수가 남으면 데이터 유지
심볼릭 링크경로 문자열을 가리키는 별도 파일원본 경로가 사라지면 깨질 수 있음
df파일시스템 전체 사용량디스크 여유 공간 확인
du파일/디렉터리별 사용량특정 폴더 용량 확인

7. 시험 함정 포인트

  • unmount가 아니라 umount입니다.
  • mkfs는 기존 데이터를 지울 수 있으므로 대상 장치를 정확히 봐야 합니다.
  • df와 du는 목적이 다릅니다.
  • LVM 순서는 PV, VG, LV입니다.
  • /etc/fstab 오류는 부팅 문제로 이어질 수 있습니다.

9. 예상문제

1. LVM 구성 순서로 알맞은 것은?

  1. ① LV → VG → PV
  2. ② PV → VG → LV
  3. ③ VG → LV → PV
  4. ④ mkfs → fdisk → PV
정답: 2. PV를 만들고 VG로 묶은 뒤 LV를 생성합니다.

2. /dev/sdb1을 /mnt/data에 연결하는 명령은?

  1. ① mkfs.ext4 /mnt/data
  2. ② mount /dev/sdb1 /mnt/data
  3. ③ umount /dev/sdb1 /mnt/data
  4. ④ fdisk /mnt/data
정답: 2. mount 장치 마운트지점 형식입니다.

10. 최종 암기 요약

inode = 파일 메타데이터
디렉터리 = 이름과 inode 연결
마운트 = 파일시스템을 디렉터리에 연결
LVM = PV → VG → LV
df -h = 전체 사용량, du -sh = 디렉터리 사용량
