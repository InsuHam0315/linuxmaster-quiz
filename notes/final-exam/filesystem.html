1. 파일시스템·디스크·마운트

디스크를 파티션으로 나누고, 파일시스템을 만들고, 디렉터리에 연결하는 전체 흐름을 초보자도 따라갈 수 있게 정리합니다.

학습 순서 1

1. 단원 개요

파일시스템은 운영체제가 파일을 저장하고 찾기 위해 사용하는 규칙입니다. 사용자는 파일 이름과 폴더만 보지만, 리눅스 내부에서는 inode, 데이터 블록, 디렉터리 항목, 마운트 지점 같은 구조가 함께 움직입니다.

디스크를 새로 추가하면 바로 파일을 저장할 수 없습니다. 일반적으로 디스크 확인 → 파티션 생성 → 파일시스템 생성 → 마운트 지점 생성 → 마운트 → 자동 마운트 설정 순서로 준비합니다. 수업 자료의 fdisk, mkfs.ext4, mount, LVM 실습이 바로 이 흐름입니다.

관련 CBT 풀기 다음: 2. 사용자 및 그룹

2. 기말고사에서 중요한 이유

  • 실습형 시험에서 새 디스크를 붙였을 때 어떤 명령을 어떤 순서로 실행하는지 자주 묻습니다.
  • df와 du, 하드 링크와 심볼릭 링크처럼 비슷해 보이는 개념을 구분해야 합니다.
  • /etc/fstab 설정 오류는 부팅 문제로 이어질 수 있어 자동 마운트 문제의 핵심입니다.
  • LVM은 PV, VG, LV의 순서를 외우는 것에서 끝나지 않고 “물리 디스크를 논리 볼륨으로 유연하게 관리한다”는 목적까지 이해해야 합니다.

3. 리눅스마스터 2급 2차 기출 연계 포인트

  • inode가 저장하는 정보: 권한, 소유자, 크기, 시간, 데이터 블록 위치. 파일명은 디렉터리 항목에 가깝습니다.
  • 하드 링크는 같은 inode를 공유하고, 심볼릭 링크는 원본 경로를 가리키는 별도 파일입니다.
  • df -h는 파일시스템 전체 사용량, du -sh는 특정 디렉터리의 실제 사용량 확인에 적합합니다.
  • mount와 umount 철자, 장치명과 마운트 지점의 순서가 자주 헷갈리는 선지로 나옵니다.
  • LVM 구성 순서는 pvcreate → vgcreate → lvcreate입니다.

4. 핵심 개념 설명

파일은 이름만으로 존재하지 않는다

리눅스에서 파일 이름은 디렉터리 안에 저장된 “이름표”에 가깝습니다. 실제 파일의 핵심 정보는 inode에 있습니다. inode는 파일의 권한, 소유자, 크기, 시간 정보, 데이터 블록 위치를 담고, 디렉터리는 파일 이름과 inode 번호를 연결합니다. 그래서 하드 링크를 만들면 이름은 두 개지만 같은 inode를 가리킬 수 있습니다.

마운트는 장치를 디렉터리 트리에 붙이는 작업

윈도우에서는 C:, D:처럼 드라이브 문자를 따로 보지만, 리눅스는 하나의 디렉터리 트리 아래에 모든 파일시스템을 붙입니다. 예를 들어 /dev/sdb1을 /mnt/data에 마운트하면, 사용자는 /mnt/data 디렉터리로 들어가 새 디스크의 내용을 사용합니다.

파일시스템 생성은 포맷과 비슷하다

mkfs.ext4 /dev/sdb1은 해당 파티션에 ext4 파일시스템 구조를 만드는 명령입니다. 기존 데이터가 사라질 수 있으므로 시험에서는 “데이터 보존”이 아니라 “파일시스템 생성”이라는 표현을 정확히 봐야 합니다.

LVM은 디스크 공간을 유연하게 관리하는 계층

LVM은 물리 디스크나 파티션을 PV로 만들고, 여러 PV를 VG로 묶은 뒤, 그 안에서 LV를 잘라 파일시스템으로 사용하는 방식입니다. 디스크를 나중에 추가하거나 볼륨 크기를 조정하기 쉬운 것이 장점입니다.

7. 시험 함정 포인트

  • umount가 정답인데 unmount를 보기로 섞는 문제가 많습니다.
  • mount /dev/sdb1 /mnt/data에서 장치와 디렉터리의 순서를 바꾼 선지를 조심합니다.
  • mkfs는 데이터를 보존하는 명령이 아니라 파일시스템을 새로 만드는 명령입니다.
  • /etc/fstab은 현재 한 번만 마운트하는 명령이 아니라 부팅 시 자동 마운트 설정 파일입니다.
  • 하드 링크는 디렉터리에 일반적으로 만들 수 없고, 다른 파일시스템을 넘기 어렵습니다. 심볼릭 링크는 파일시스템 경계를 넘을 수 있습니다.
  • LVM 순서는 PV → VG → LV입니다. “LV를 먼저 만든다”는 선지는 틀립니다.

8. 자주 나오는 선지 표현

  • “디렉터리별 실제 사용량을 확인한다” → du -sh
  • “마운트된 파일시스템의 여유 공간을 확인한다” → df -h
  • “부팅 시 자동 마운트 정보를 저장한다” → /etc/fstab
  • “같은 inode를 공유한다” → 하드 링크
  • “원본 경로를 가리키는 별도 파일이다” → 심볼릭 링크
  • “LVM에서 물리 볼륨을 먼저 생성한다” → pvcreate

9. 예상문제

1. 새 파티션 /dev/sdb1에 ext4 파일시스템을 만든 뒤 /data에 연결하는 순서로 가장 적절한 것은?

  1. mount → mkfs.ext4 → mkdir
  2. mkfs.ext4 → mkdir → mount
  3. mkdir → umount → fsck
  4. df → du → blkid
정답: 2. 파일시스템을 만들고, 마운트 지점을 만든 뒤, 장치를 디렉터리에 마운트합니다.

2. 하드 링크에 대한 설명으로 옳은 것은?

  1. 원본 경로 문자열만 저장한다.
  2. 항상 다른 파일시스템에 만들 수 있다.
  3. 같은 inode를 공유하고 링크 수가 증가한다.
  4. 원본이 삭제되면 무조건 모든 데이터가 사라진다.
정답: 3. 하드 링크는 같은 inode를 가리키는 다른 이름입니다.

3. LVM 구성 순서로 알맞은 것은?

  1. LV → VG → PV
  2. VG → PV → LV
  3. PV → VG → LV
  4. mkfs → LV → PV
정답: 3. 물리 볼륨(PV)을 만들고, 볼륨 그룹(VG)으로 묶고, 논리 볼륨(LV)을 생성합니다.

10. 최종 암기 요약

  • 디스크 준비 흐름: 확인 → 파티션 → 파일시스템 → 마운트 지점 → 마운트 → fstab.
  • df는 전체 파일시스템, du는 특정 파일/디렉터리 사용량.
  • 하드 링크는 같은 inode, 심볼릭 링크는 경로를 가리키는 별도 파일.
  • LVM은 PV → VG → LV, 그 뒤 mkfs와 mount.
  • /etc/fstab은 자동 마운트 설정 파일이다.
