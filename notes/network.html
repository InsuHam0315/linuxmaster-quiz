네트워크 및 서비스

IP, 서브넷, 게이트웨이, DNS, TCP/UDP, 주요 포트와 네트워크 명령을 정리합니다.

시험에 나오는 핵심

  • IP 주소, 서브넷, 게이트웨이, DNS의 역할을 구분합니다.
  • TCP는 연결지향, UDP는 비연결지향이라는 키워드가 중요합니다.
  • 서비스 이름과 포트 번호는 빠르게 매칭해야 합니다.

핵심 개념 정리

항목정리
IP 주소네트워크에서 호스트를 식별하는 주소입니다.
서브넷네트워크 영역을 나누는 단위입니다. 마스크와 함께 판단합니다.
게이트웨이다른 네트워크로 나갈 때 거치는 경로입니다.

pingICMP 기반으로 연결 가능 여부를 확인합니다.
ip addr, ip route주소와 라우팅 정보를 확인합니다.

TCP / UDPTCP는 연결지향, UDP는 비연결지향입니다.

시험 함정 포인트

  • SSH는 22번, Telnet은 23번입니다.
  • HTTP는 80, HTTPS는 443입니다.
  • DNS는 53번입니다.
  • ifconfig는 구식 명령이지만 시험 선택지에는 나올 수 있습니다.
  • ping은 ICMP 기반입니다.

1분 암기 요약

  • 22 SSH, 23 Telnet, 53 DNS, 80 HTTP, 443 HTTPS.
  • TCP=연결지향, UDP=비연결지향.
  • ip addr=주소, ip route=경로, ss=소켓 확인.

처음 공부하는 사람용 흐름

1. 주소 체계

IP 주소는 내 위치, 서브넷은 같은 네트워크 범위, 게이트웨이는 밖으로 나가는 문, DNS는 이름을 IP로 바꾸는 역할입니다.

2. 통신 방식

TCP는 연결을 확인하며 안정성을 중시하고, UDP는 연결 확인 없이 빠르게 보냅니다. 시험에서는 “연결지향/비연결지향”으로 출제됩니다.

3. 확인 명령

ip addr로 주소, ip route로 경로, ping으로 연결성, ss로 포트·소켓 상태를 확인합니다.

4. 포트 번호

서비스 이름과 포트를 바로 연결해야 합니다. SSH 22, DNS 53, HTTP 80, HTTPS 443은 반드시 즉시 떠올라야 합니다.

기출형 문제 풀이

문제. 다음 중 포트 번호 연결이 옳지 않은 것은?

  1. SSH - 22
  2. DNS - 53
  3. HTTP - 80
  4. HTTPS - 110
정답: 4번

HTTPS는 443입니다. 110은 POP3입니다. HTTP 80, SSH 22, DNS 53은 필수 암기 포트입니다.
