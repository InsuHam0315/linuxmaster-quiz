네트워크 및 서비스

IP, 서브넷, 게이트웨이, DNS, TCP/UDP, 주요 포트와 네트워크 명령을 정리합니다.

시험에 나오는 핵심

  • IP 주소, 서브넷, 게이트웨이, DNS의 역할을 구분합니다.
  • TCP는 연결지향, UDP는 비연결지향이라는 키워드가 중요합니다.
  • 서비스 이름과 포트 번호는 빠르게 매칭해야 합니다.

강의식 개념노트

네트워크는 주소와 길 찾기입니다

IP 주소는 컴퓨터의 네트워크상 주소입니다. 서브넷 마스크는 같은 동네인지 다른 동네인지 판단하는 기준입니다. 게이트웨이는 다른 네트워크로 나갈 때 거치는 출입문입니다. DNS는 사람이 읽는 도메인 이름을 IP 주소로 바꿔 줍니다.

TCP와 UDP는 통신 성격이 다릅니다

TCP는 연결을 맺고 확인하면서 데이터를 보내는 연결지향 방식입니다. 신뢰성이 필요한 웹, SSH, 메일 같은 서비스에 많이 연결됩니다. UDP는 연결 확인 없이 빠르게 보내는 비연결지향 방식이며 DNS 질의처럼 짧고 빠른 통신과 자주 연결됩니다.

포트 번호는 서비스의 문 번호입니다

서버 한 대 안에서도 여러 서비스가 동시에 동작할 수 있습니다. 포트 번호는 그중 어떤 서비스로 들어갈지 구분하는 번호입니다. SSH 22, Telnet 23, DNS 53, HTTP 80, HTTPS 443은 바로 떠올릴 수 있어야 합니다.

네트워크 명령은 장애 지점을 찾는 도구입니다

ip addr로 내 주소를 보고, ip route로 나가는 길을 확인하고, ping으로 연결 가능성을 확인합니다. ss나 netstat는 서비스가 포트를 열고 기다리는지 볼 때 사용합니다.

명령어 해설 노트

연결 확인

ping은 ICMP로 상대 호스트와 통신이 되는지 확인합니다. traceroute나 tracepath는 목적지까지 어떤 라우터를 거치는지 추적합니다.

주소와 라우팅

ip addr는 인터페이스와 IP 주소를 확인합니다. ip route는 기본 게이트웨이와 라우팅 경로를 봅니다. ifconfig는 예전 명령이지만 시험 선택지에 나올 수 있습니다.

포트와 소켓

ss와 netstat는 열려 있는 포트, 연결 상태, 대기 중인 소켓을 확인합니다. 서비스가 실행 중인지 볼 때 포트 번호와 함께 읽습니다.

원격 접속·전송

ssh는 암호화된 원격 접속이고 기본 포트는 22입니다. telnet은 암호화되지 않습니다. ftp는 파일 전송, sftp는 SSH 기반 파일 전송입니다.

시험 함정 포인트

  • SSH는 22번, Telnet은 23번입니다.
  • HTTP는 80, HTTPS는 443입니다.
  • DNS는 53번입니다.
  • ifconfig는 구식 명령이지만 시험 선택지에는 나올 수 있습니다.
  • ping은 ICMP 기반입니다.

1분 암기 요약

  • 22 SSH, 23 Telnet, 53 DNS, 80 HTTP, 443 HTTPS.
  • TCP=연결지향, UDP=비연결지향.
  • ip addr=주소, ip route=경로, ss=소켓 확인.

처음 공부하는 사람용 흐름

1. 주소 체계

IP 주소는 내 위치, 서브넷은 같은 네트워크 범위, 게이트웨이는 밖으로 나가는 문, DNS는 이름을 IP로 바꾸는 역할입니다.

2. 통신 방식

TCP는 연결을 확인하며 안정성을 중시하고, UDP는 연결 확인 없이 빠르게 보냅니다. 시험에서는 “연결지향/비연결지향”으로 출제됩니다.

3. 확인 명령

ip addr로 주소, ip route로 경로, ping으로 연결성, ss로 포트·소켓 상태를 확인합니다.

4. 포트 번호

서비스 이름과 포트를 바로 연결해야 합니다. SSH 22, DNS 53, HTTP 80, HTTPS 443은 반드시 즉시 떠올라야 합니다.

기출형 문제 풀이

문제. 다음 중 포트 번호 연결이 옳지 않은 것은?

  1. SSH - 22
  2. DNS - 53
  3. HTTP - 80
  4. HTTPS - 110
정답: 4번

HTTPS는 443입니다. 110은 POP3입니다. HTTP 80, SSH 22, DNS 53은 필수 암기 포트입니다.
