프로세스 및 작업 관리

PID, foreground/background, 작업 제어, signal, nice, nohup, cron과 at을 정리합니다.

시험에 나오는 핵심

  • 프로세스는 실행 중인 프로그램이며 PID로 식별합니다.
  • 작업 제어는 &, jobs, fg, bg, Ctrl 키 조합을 함께 봅니다.
  • signal은 종료 방식 차이를 묻는 단골 영역입니다.

강의식 개념노트

프로그램과 프로세스는 다릅니다

프로그램은 디스크에 저장된 실행 파일이고, 프로세스는 그 프로그램이 메모리에 올라와 실행 중인 상태입니다. 같은 프로그램을 여러 번 실행하면 PID가 다른 여러 프로세스가 생길 수 있습니다. 시험에서는 PID, PPID, 부모·자식 관계를 묻습니다.

전면 작업과 후면 작업

foreground 작업은 터미널을 붙잡고 있어 작업이 끝나기 전까지 다른 명령을 치기 어렵습니다. background 작업은 뒤에서 실행되고, 명령 뒤에 &를 붙이는 식으로 실행합니다. 멈춘 작업은 fg, bg, jobs로 관리합니다.

종료는 signal로 이해합니다

프로세스를 끝낼 때는 단순히 “죽인다”가 아니라 signal을 보냅니다. 기본 kill은 SIGTERM으로 정상 종료를 요청합니다. kill -9는 SIGKILL로 강제 종료입니다. 시험에서는 SIGTERM과 SIGKILL을 바꿔 내는 경우가 많습니다.

예약과 우선순위는 운영 관리 영역입니다

cron은 매일, 매주처럼 반복 실행할 작업을 예약합니다. at은 한 번만 실행할 작업입니다. nice 값은 CPU 우선순위와 관련 있고, 값이 낮을수록 우선순위가 높다는 점을 기억하세요.

명령어 해설 노트

ps -ef와 ps aux

둘 다 프로세스 목록을 봅니다. ps -ef는 System V 스타일로 UID, PID, PPID 같은 열을 보기 좋고, ps aux는 BSD 스타일로 CPU·메모리 사용률 확인에 자주 연결됩니다.

jobs, fg, bg

현재 셸에서 실행한 작업을 관리합니다. jobs는 작업 목록, fg는 foreground로 가져오기, bg는 background에서 계속 실행시키기입니다.

kill과 signal

kill PID는 기본적으로 SIGTERM을 보내 정상 종료를 요청합니다. kill -9 PID는 SIGKILL이라 강제 종료입니다.

예약·우선순위

nice는 실행할 때 우선순위를 조정하고, renice는 실행 중인 프로세스의 nice 값을 바꿉니다. cron은 반복 예약, at은 한 번만 실행하는 예약입니다.

헷갈리는 비교표

비교핵심 차이
cron / atcron은 반복 예약, at은 1회성 예약입니다.
SIGTERM정상 종료를 요청합니다. kill의 기본 signal입니다.
SIGKILL강제 종료입니다. 잡거나 무시할 수 없습니다.
SIGHUP터미널 연결 종료나 설정 재읽기 상황과 연결되어 출제됩니다.

시험 함정 포인트

  • kill 기본 signal은 SIGTERM입니다.
  • Ctrl+Z는 일시정지, Ctrl+C는 인터럽트입니다.
  • &는 백그라운드 실행입니다.
  • ps -ef는 System V 스타일, ps aux는 BSD 스타일로 자주 구분합니다.

1분 암기 요약

  • PID=내 번호, PPID=부모 번호.
  • SIGTERM=기본 종료 요청, SIGKILL=무조건 강제 종료.
  • cron=반복, at=한 번, nohup=로그아웃 후 유지.

처음 공부하는 사람용 흐름

1. 프로세스 식별

실행 중인 프로그램은 PID로 구분합니다. 부모 프로세스는 PPID로 보며, 트리 구조를 생각하면 됩니다.

2. 확인 명령

ps는 순간 목록, top은 실시간 확인입니다. ps aux와 ps -ef는 형식 차이로 자주 출제됩니다.

3. 작업 제어

&는 백그라운드 실행, Ctrl+Z는 일시정지, fg/bg는 전면·후면 전환입니다.

4. 종료 신호

먼저 SIGTERM으로 정상 종료를 요청하고, 안 되면 SIGKILL을 씁니다. SIGKILL은 잡거나 무시할 수 없습니다.

기출형 문제 풀이

문제. 다음 중 설명이 옳은 것은?

  1. kill의 기본 signal은 SIGKILL이다.
  2. Ctrl+C는 현재 작업을 일시정지한다.
  3. nohup은 로그아웃 후에도 명령 실행을 유지할 때 사용한다.
  4. cron은 1회성 예약 작업에만 사용한다.
정답: 3번
