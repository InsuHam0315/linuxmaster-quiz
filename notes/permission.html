권한·소유권·특수권한

rwx, user/group/others, chmod, chown, umask와 특수권한 숫자 계산을 정리합니다.

시험에 나오는 핵심

  • 권한은 user, group, others 순서로 읽습니다.
  • 숫자 권한은 r=4, w=2, x=1의 합입니다.
  • 특수권한은 일반 3자리 앞에 4, 2, 1 조합으로 붙습니다.

강의식 개념노트

권한은 “누가 무엇을 할 수 있나”입니다

리눅스 권한은 소유자(user), 그룹(group), 기타 사용자(others) 세 묶음으로 봅니다. 각 묶음 안에서 읽기(r), 쓰기(w), 실행(x)을 판단합니다. rwxr-xr--처럼 나오면 세 글자씩 끊어서 rwx, r-x, r--로 읽는 습관을 들이면 됩니다.

숫자 권한은 더하기 문제입니다

r=4, w=2, x=1입니다. rwx는 4+2+1=7, rw-는 4+2=6, r-x는 4+1=5입니다. 그래서 chmod 754 file은 user=rwx, group=r-x, others=r--입니다.

디렉터리 권한은 파일과 의미가 다릅니다

파일의 x는 실행 가능 여부입니다. 디렉터리의 x는 그 디렉터리에 들어가거나 하위 파일에 접근할 수 있는 탐색 권한입니다. 디렉터리에 w가 있으면 그 안에서 파일 생성·삭제·이름 변경이 가능하다는 식으로 출제됩니다.

특수권한은 앞자리만 보면 됩니다

chmod 1777 /tmp처럼 네 자리 권한이 나오면 맨 앞자리가 특수권한입니다. 4는 setuid, 2는 setgid, 1은 sticky bit입니다. sticky bit는 공유 디렉터리에서 남의 파일 삭제를 제한하는 개념으로 자주 나옵니다.

핵심 개념 정리

항목정리
r파일은 읽기, 디렉터리는 목록 조회 권한입니다.
w파일은 수정, 디렉터리는 생성·삭제·이름 변경과 관련됩니다.
x파일은 실행, 디렉터리는 접근·탐색 권한입니다.
chmod숫자 모드 chmod 755 file, 기호 모드 chmod u+x file처럼 사용합니다.
chown, chgrp파일 소유자와 그룹을 변경합니다.
umask새 파일·디렉터리 생성 시 기본 권한에서 빼는 마스크입니다.

자주 나오는 명령어/키워드

  • chmod 755
  • chmod u+rwx
  • chown user file
  • chown user:group file
  • chgrp group file
  • umask
  • setuid
  • setgid
  • sticky bit

명령어 해설 노트

chmod 755 file

세 자리는 user, group, others 순서입니다. 7은 rwx, 5는 r-x이므로 소유자는 모든 권한, 그룹과 기타 사용자는 읽기·실행 권한입니다.

chmod u+rwx file

기호 방식은 누구에게 어떤 권한을 더하거나 빼는지 표현합니다. u/g/o/a는 대상, +/-/=는 추가·제거·정확히 지정을 뜻합니다.

chown과 chgrp

chown user file은 소유자 변경, chown user:group file은 소유자와 그룹 동시 변경, chgrp group file은 그룹만 변경입니다.

umask와 특수권한

umask는 새 파일·디렉터리의 기본 권한에서 제거할 권한입니다. 특수권한은 네 자리 권한의 앞자리이며 4=setuid, 2=setgid, 1=sticky bit입니다.

헷갈리는 비교표

권한숫자의미예
setuid4실행 시 파일 소유자 권한으로 실행4770, 5770
setgid2실행 시 그룹 권한, 디렉터리는 하위 파일 그룹 상속2770, 3770
sticky bit1공유 디렉터리에서 소유자 중심 삭제 제한1770, 3770
조합합산4+2+1처럼 더해서 앞자리에 표시3770=setgid+sticky, 5770=setuid+sticky

시험 함정 포인트

  • 파일의 x는 실행, 디렉터리의 x는 접근·탐색입니다.
  • sticky bit는 공유 디렉터리에서 남의 파일 삭제를 제한하는 쪽으로 자주 출제됩니다.
  • 777 앞에 특수권한 숫자가 붙어 1777처럼 나올 수 있습니다.
  • 4=setuid, 2=setgid, 1=sticky 순서를 바꾸지 마세요.

1분 암기 요약

  • r=4, w=2, x=1. user/group/others 순서.
  • 특수권한 앞자리: 4 setuid, 2 setgid, 1 sticky.
  • 디렉터리 x는 들어갈 수 있는 권한입니다.

처음 공부하는 사람용 흐름

1. 3칸으로 끊기

rwxr-xr--는 rwx, r-x, r--로 끊어 user/group/others 순서로 읽습니다.

2. 숫자로 바꾸기

각 칸에서 r=4, w=2, x=1을 더합니다. rwx는 7, r-x는 5, rw-는 6입니다.

3. 디렉터리 권한

디렉터리는 x가 있어야 들어가거나 하위 파일에 접근할 수 있습니다. 파일 실행 권한과 의미가 다릅니다.

4. 특수권한 앞자리

chmod 1777처럼 네 자리면 맨 앞자리가 특수권한입니다. 4, 2, 1을 더해서 조합을 판단합니다.

기출형 문제 풀이

문제. chmod 2750 project에 대한 설명으로 옳은 것은?

  1. setuid가 설정된다.
  2. setgid가 설정되고 일반 권한은 rwxr-x---이다.
  3. sticky bit가 설정되고 일반 권한은 rw-r-----이다.
  4. others에게 읽기 권한이 있다.
정답: 2번

앞자리 2는 setgid입니다. 뒤의 750은 user=rwx, group=r-x, others=---입니다.
