셸·환경변수·스크립트

셸 종류, 변수, export, PATH, alias, bash 시작 파일과 스크립트 기본 문법을 정리합니다.

시험에 나오는 핵심

  • 셸은 사용자 명령을 해석해 커널에 전달하는 명령 해석기입니다.
  • 지역변수와 환경변수의 차이는 자식 프로세스 전달 여부입니다.
  • bash 시작 파일은 login shell 여부로 자주 출제됩니다.

강의식 개념노트

셸은 명령어 통역사입니다

사용자가 ls나 cp를 입력하면 셸이 그 문장을 해석하고 실제 프로그램을 실행합니다. 그래서 셸 종류, 환경변수, alias, 스크립트 문법은 명령 실행 방식에 직접 영향을 줍니다.

변수는 전달 범위가 핵심입니다

지역변수는 현재 셸 안에서만 주로 사용됩니다. 환경변수는 export되어 자식 프로세스에도 전달됩니다. “자식 프로세스가 사용할 수 있게 한다”라는 표현이 나오면 export를 떠올리면 됩니다.

PATH는 명령어를 찾는 길입니다

명령을 칠 때 매번 /bin/ls처럼 전체 경로를 쓰지 않아도 되는 이유가 PATH입니다. PATH 안에 들어 있는 디렉터리를 순서대로 뒤져 실행 파일을 찾습니다. 현재 디렉터리 자체를 의미하는 값이 아니라는 점이 중요합니다.

스크립트는 명령어를 파일로 묶은 것입니다

셸 스크립트는 사람이 터미널에 치던 명령어를 파일에 순서대로 적어 자동 실행하는 방식입니다. 첫 줄의 #!/bin/bash는 이 파일을 어떤 셸로 해석할지 알려주는 표시입니다.

명령어 해설 노트

export

NAME=value처럼 만든 변수는 현재 셸에서만 보입니다. export NAME 또는 export NAME=value를 해야 자식 프로세스도 값을 볼 수 있습니다.

PATH

PATH는 명령어 실행 파일을 찾는 디렉터리 목록입니다. 현재 위치를 뜻하는 변수는 PWD이고, PATH와 다릅니다.

alias

자주 쓰는 명령 조합에 별명을 붙입니다. alias ll='ls -l'처럼 설정할 수 있으며, 새 프로그램 설치가 아니라 셸이 명령을 치환하는 기능입니다.

bash 시작 파일

/etc/profile은 시스템 전체 로그인 셸 설정, ~/.bash_profile은 사용자 로그인 셸 설정, ~/.bashrc는 대화형 non-login bash 설정과 연결합니다.

헷갈리는 비교표

비교핵심 차이
지역변수현재 셸에서만 주로 사용합니다. 자식 프로세스에 자동 전달되지 않습니다.
환경변수export된 변수로 자식 프로세스에서도 사용할 수 있습니다.
.bash_profile사용자 login shell에서 주로 읽습니다.
.bashrc사용자 interactive non-login bash에서 주로 읽습니다.
/etc/profile시스템 전체 login shell 설정에 사용됩니다.

시험 함정 포인트

  • export를 해야 자식 프로세스에 전달됩니다.
  • PATH는 명령어 탐색 경로이지 현재 디렉터리 표시 변수가 아닙니다.
  • C Shell은 csh입니다.
  • .bashrc와 .bash_profile의 읽히는 상황을 구분하세요.

1분 암기 요약

  • shell=명령 해석기, bash=대표 GNU 셸.
  • export=자식에게 전달, PATH=명령 찾는 길.
  • login은 .bash_profile, interactive non-login은 .bashrc.

처음 공부하는 사람용 흐름

1. 셸의 역할

사용자가 입력한 명령을 해석하고 프로그램을 실행해 주는 중간 계층입니다. 그래서 셸 문법과 환경변수가 명령 실행 결과를 바꿉니다.

2. 변수 구분

현재 셸에서만 쓰는 값은 지역변수, 자식 프로세스까지 넘기는 값은 환경변수입니다. 시험에서는 export 여부를 묻습니다.

3. 시작 파일

로그인 셸은 .bash_profile, 대화형 non-login bash는 .bashrc를 중심으로 기억합니다.

4. 스크립트 흐름

#!/bin/bash로 실행 셸을 지정하고, 조건문 if, 반복문 for/while, 분기 case를 사용합니다.

기출형 문제 풀이

문제. 환경변수와 bash 시작 파일에 대한 설명으로 옳은 것은?

  1. PATH는 현재 작업 디렉터리만 저장한다.
  2. export를 해야 변수가 자식 프로세스에 전달될 수 있다.
  3. .bashrc는 커널 부팅 직후 항상 가장 먼저 실행된다.
  4. C Shell의 명령 이름은 bash이다.
