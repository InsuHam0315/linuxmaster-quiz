리눅스 일반

커널, GNU/Linux, 배포판, CLI/GUI, 주요 디렉터리와 GRUB을 시험 관점으로 정리합니다.

시험에 나오는 핵심

  • Linux는 운영체제 전체보다 커널 중심 개념으로 묻는 경우가 많습니다.
  • GNU 도구와 Linux 커널이 결합된 환경을 GNU/Linux라고 부릅니다.
  • 배포판 계열과 기본 패키지 관리 도구를 함께 외우면 다른 과목도 편합니다.

핵심 개념 정리

개념정리
커널하드웨어 제어, 프로세스·메모리·파일시스템·장치 관리를 담당하는 핵심 부분입니다.
오픈소스/자유 소프트웨어소스 공개와 사용·수정·재배포 자유를 중시합니다. 라이선스 조건은 각각 다를 수 있습니다.
배포판커널, GNU 도구, 설치 도구, 패키지 저장소, 데스크톱 환경 등을 묶어 제공한 운영체제 형태입니다.
CLI/GUICLI는 명령어 기반, GUI는 그래픽 기반 조작 방식입니다.
GRUB부팅 시 커널을 선택하고 로드하는 대표 부트로더입니다.

헷갈리는 비교표

항목시험용 구분
Debian 계열Debian, Ubuntu. 보통 dpkg, apt와 연결됩니다.
Red Hat 계열RHEL, CentOS, Rocky Linux. 보통 rpm, yum, dnf와 연결됩니다.
Slackware 계열전통적인 단순 구조를 강조하는 계열로 분류 문제에 나옵니다.
/etc vs /home/etc는 설정 파일, /home은 일반 사용자 홈 디렉터리입니다.

시험 함정 포인트

  • Ubuntu는 Red Hat 계열이 아니라 Debian 계열입니다.
  • Rocky Linux, RHEL, CentOS는 Red Hat 계열입니다.
  • /dev는 장치 파일, /var는 로그·스풀처럼 변하는 데이터가 핵심입니다.
  • /bin은 일반 필수 명령, /sbin은 관리자용 필수 명령과 연결됩니다.

1분 암기 요약

  • Linux=커널, GNU/Linux=커널+GNU 도구.
  • Debian=Ubuntu=apt, Red Hat=RHEL/Rocky=dnf/rpm.
  • /etc 설정, /home 사용자, /var 변동, /dev 장치.

처음 공부하는 사람용 흐름

1. 리눅스의 정체

시험에서는 Linux를 “커널”로 보는지, 배포판 전체로 보는지 구분시키는 문제가 많습니다. Ubuntu 같은 배포판은 커널에 명령어, 설치 도구, 패키지 저장소를 묶은 결과입니다.

2. 디렉터리 먼저 외우기

명령어보다 먼저 /etc, /home, /var, /dev의 역할을 잡으면 로그, 설정, 장치 문제를 빠르게 풀 수 있습니다.

3. 계열과 패키지 연결

Debian 계열은 apt, Red Hat 계열은 rpm/dnf로 연결해서 외우세요. 패키지 과목 문제와 함께 반복 출제됩니다.

4. 부팅 키워드

GRUB은 운영체제를 고르는 메뉴가 아니라 커널을 메모리에 올리는 부트로더라는 점이 핵심입니다.

기출형 문제 풀이

문제. 다음 중 설명이 옳은 것은?

  1. Linux는 GNU 프로젝트에서 만든 셸 이름이다.
  2. Ubuntu는 Red Hat 계열 배포판이다.
  3. /etc는 시스템 설정 파일이 주로 위치하는 디렉터리이다.
  4. /home은 장치 파일이 저장되는 디렉터리이다.
정답: 3번

Linux는 커널 중심 개념이고 Ubuntu는 Debian 계열입니다. 장치 파일은 /dev, 사용자 홈은 /home, 설정 파일은 /etc로 연결합니다.
