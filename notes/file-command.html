파일 및 디렉터리 명령어

파일 탐색, 조작, 검색, 필터, 리다이렉션, 파이프와 링크를 문제 선택지 기준으로 정리합니다.

시험에 나오는 핵심

  • 기본 파일 명령은 옵션 의미까지 묻습니다.
  • 리다이렉션은 표준출력과 표준에러 구분이 핵심입니다.
  • 하드 링크와 심볼릭 링크는 inode와 원본 삭제 상황으로 구분합니다.

강의식 개념노트

파일 명령은 네 단계로 정리합니다

처음에는 명령어를 무작정 외우지 말고 “위치 확인 → 파일 조작 → 내용 확인 → 검색·가공” 순서로 묶으세요. 위치 확인은 pwd, cd, ls입니다. 파일 조작은 touch, mkdir, cp, mv, rm입니다. 내용 확인은 cat, less, head, tail입니다.

옵션은 상황으로 이해합니다

ls -al은 숨김 파일까지 자세히 보는 상황, cp -r은 디렉터리를 통째로 복사하는 상황, rm -r은 디렉터리 안쪽까지 삭제하는 상황입니다. 시험은 “디렉터리를 복사하려면?”, “숨김 파일까지 보려면?”처럼 상황을 주고 옵션을 고르게 합니다.

리다이렉션과 파이프는 흐름입니다

명령어 결과가 화면으로 가는 대신 파일로 가면 리다이렉션입니다. >는 덮어쓰기, >>는 추가입니다. 파이프 |는 앞 명령의 결과를 다음 명령의 재료로 넘깁니다. 예를 들어 목록을 뽑고 그 안에서 다시 검색하는 식의 조합 문제가 나옵니다.

링크는 원본과의 관계가 핵심입니다

하드 링크는 같은 파일 실체를 다른 이름으로 가리키는 느낌이고, 심볼릭 링크는 원본 경로를 적어 둔 바로가기 파일에 가깝습니다. 그래서 심볼릭 링크는 원본이 사라지면 깨질 수 있습니다. inode, 원본 삭제, 다른 파일시스템 여부가 시험 포인트입니다.

명령어 해설 노트

파일 찾기: find

find . -name "*.log"는 현재 디렉터리 아래에서 이름이 .log로 끝나는 파일을 찾습니다. -type f는 일반 파일, -type d는 디렉터리만 찾습니다.

내용 찾기: grep

grep은 파일 이름이 아니라 파일 내용에서 문자열을 찾습니다. -i는 대소문자 무시, -n은 줄 번호 표시, -v는 조건에 맞지 않는 줄 출력입니다.

출력 저장: 리다이렉션

>는 덮어쓰기, >>는 추가입니다. 2>는 표준에러만 저장하고, 2>&1은 표준에러를 표준출력과 같은 곳으로 합칩니다.

연결과 링크

파이프 |는 앞 명령 결과를 뒤 명령 입력으로 넘깁니다. ln file hard는 하드 링크, ln -s file soft는 심볼릭 링크입니다.

헷갈리는 비교표

항목차이
> / >>>는 덮어쓰기, >>는 뒤에 추가입니다.
rm -r / rm -f-r은 하위까지 재귀 삭제, -f는 확인·오류 메시지를 줄이는 강제 처리입니다.
하드 링크같은 inode를 가리키며 일반적으로 디렉터리나 다른 파일시스템에는 제한됩니다.
심볼릭 링크경로를 가리키는 별도 파일입니다. 원본이 사라지면 깨진 링크가 될 수 있습니다.

시험 함정 포인트

  • 2>&1은 표준에러를 표준출력 쪽으로 합칩니다.
  • grep -v는 매칭되지 않는 줄 출력입니다.
  • find -exec 명령 {} \; 형태의 의미를 자주 묻습니다.
  • 파이프 |는 앞 명령의 표준출력을 다음 명령의 표준입력으로 넘깁니다.

1분 암기 요약

  • cp -r 복사, rm -r 삭제, tail -f 실시간 확인.
  • grep -i 대소문자 무시, -n 줄 번호, -v 반전.
  • > 덮기, >> 추가, 2>&1 에러 합치기.

처음 공부하는 사람용 흐름

1. 위치 확인

pwd로 현재 위치를 보고, cd로 이동하고, ls로 목록을 확인합니다. 파일 명령은 이 세 개가 출발점입니다.

2. 생성·복사·삭제

mkdir, touch, cp, mv, rm은 옵션 함정이 많습니다. 디렉터리는 대부분 -r이 필요하다고 기억하세요.

3. 내용 보기와 검색

짧으면 cat, 길면 less, 앞부분은 head, 끝부분은 tail, 실시간 로그는 tail -f입니다.

4. 조합 문제

시험은 find, grep, 파이프, 리다이렉션을 섞어 냅니다. 한 명령의 출력이 다음 명령의 입력이 되는 구조를 먼저 읽으세요.

기출형 문제 풀이

문제. grep -in "error" app.log의 설명으로 옳은 것은?

  1. error가 없는 줄만 출력한다.
  2. 대소문자를 무시하고 error를 찾으며 줄 번호도 함께 출력한다.
  3. app.log의 마지막 10줄만 출력한다.
  4. 검색 결과를 app.log에 추가 저장한다.
정답: 2번

-i는 대소문자 무시, -n은 줄 번호 출력입니다. 반전 검색은 -v, 마지막 줄 확인은 tail입니다.
